# Selection Guide

Use observable interface needs instead of aesthetic adjectives alone.

## Selection Map

| Interface need | Primary choices |
|---|---|
| Soft first impression | Fade & Blur |
| Direction or navigation continuity | Directional Slide |
| Editorial image or headline unveiling | Mask Reveal |
| Long-page reading rhythm | Scroll Reveal |
| Layered spatial depth | Parallax Depth |
| Quiet hero atmosphere | Float & Levitate |
| Continuous inventory or brand rhythm | Marquee |
| Process, route, or signal connection | Line Drawing |
| Meaningful metric change | Number Ticker |
| Live generation or terminal-like response | Typewriter |
| Living system or relationship map | Orbit Network |
| Material highlight or loading surface | Shimmer Sweep |

## Composition Rules

1. Select one primary motion for the surface.
2. Add at most one supporting motion.
3. Pair a structural motion with an ambient or polish motion; do not stack two structural entrances on the same object.
4. Run only one continuous attention-seeking loop in a viewport.
5. Keep essential reading content static after arrival.
6. Let user selection override automatic selection.

Safe examples:

- Fade & Blur + Number Ticker
- Scroll Reveal + Line Drawing
- Mask Reveal + Shimmer Sweep
- Parallax Depth + Float & Levitate

Avoid:

- Marquee + Orbit Network in the same focal region
- Parallax Depth on long reading copy
- Typewriter for full paragraphs
- Shimmer Sweep on every card
- Scroll Reveal on every icon, label, and divider

## Automatic Mode

When the user delegates selection:

1. Match the target trigger and intent to the catalog.
2. Prefer the lowest-intensity pattern that communicates the relationship.
3. Preserve the existing project stack and interaction model.
4. Return the selected ID, editable parameters, and finished result.
5. Do not output internal scoring or chain-of-thought.
